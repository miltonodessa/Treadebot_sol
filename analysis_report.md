# АНАЛИЗ СТРАТЕГИИ nya666 - ПОЛНЫЙ ОТЧЕТ

## 1. ЧТО МЫ ВИДИМ ИЗ ТЕКУЩИХ ДАННЫХ

### Стратегия - это НЕ снайпер "купил-продал". Это АКТИВНЫЙ ТРЕЙДЕР

Пример токена `8datMCrTHUKEV22YBV58`:
- Buy 1: +0s   → 0.099 SOL при BC=1.58 SOL  (очень ранняя покупка!)
- Buy 2: +124s  → 0.100 SOL при BC=5.47 SOL  (+246% рост BC)
- Buy 3: +475s  → 0.196 SOL при BC=10.60 SOL
- Buy 13: +9263s → 0.642 SOL при BC=21.21 SOL (+1239% от старта!)
- 13 покупок + 13 продаж на одном токене за 2.5 часа

### Реальная схема торговли:
1. **Probe buy** (~0.09-0.10 SOL) при BC = 1-5 SOL (очень ранняя точка входа)
2. **Scale in** (+0.10-0.20 SOL) если BC растет
3. **Sell часть** → фиксация прибыли
4. **Re-buy** если BC остается в диапазоне
5. Повтор 10-13 раз пока токен живет

### Конкретные числа:
- Медиана времени между повторными покупками: 180 секунд
- Токены с 5+ покупками: 157
- 640 случаев averaging down (покупка при падении BC)
- 370 случаев averaging up (покупка при росте BC >100%)

## 2. ЧТО ОПРЕДЕЛЯЕТ РАЗМЕР ПОЗИЦИИ

Тиры (не случайные - это УРОВНИ УВЕРЕННОСТИ):
```
0.09-0.10 SOL → Tier 1 (probe, самый первый вход)
0.19-0.21 SOL → Tier 2 (подтвержденный сигнал)
0.29-0.32 SOL → Tier 3 (сильный momentum)
0.39-0.42 SOL → Tier 4 (очень сильный сигнал)
0.50-0.60 SOL → Tier 5
0.76-0.81 SOL → Tier 6 (максимальная конвикция)
3.00+ SOL     → Tier 7 (редкий, исключительный сигнал)
```

**ВЫВОД**: BC pre-SOL НЕ определяет размер (медиана одинакова ~10 SOL для всех тиров).
Значит размер определяется ВНЕШНИМИ СИГНАЛАМИ которые мы не видим.

## 3. ЧТО ОПРЕДЕЛЯЕТ ВЫБОР ТОКЕНА (НЕ ВИДНО ИЗ ДАННЫХ)

Нам нужно собрать:
- Время от создания токена до первой покупки nya666
- Dev initial buy (сколько SOL вложил создатель)
- Кто купил до nya666 (другие известные боты/кошельки)
- Метаданные токена (есть ли сайт/twitter при создании)

## 4. ЧТО НУЖНО ПАРСИТЬ

### Parser 1: Token Creation Data
Для каждого mint → найти CREATE транзакцию через Helius API
Получить:
- block_time создания (→ delta_t до первой покупки nya666)
- dev_buy_amount (сколько SOL дев купил при создании)
- Есть ли metadata (uri → проверить siteUrl, twitter)

### Parser 2: Early Buyers Data
Для каждого mint → все транзакции от создания до +60 секунд
Получить:
- n_buyers_before_nya666 (сколько купили до него)
- known_wallets_bought (известные умные кошельки)
- buy_pressure (скорость накопления SOL в кривой)

### Parser 3: Price Action During Hold
Для каждого mint → все BC changes во время позиции nya666
Получить:
- max_bc_during_hold (максимальный BC пока держал)
- exit_bc (BC при продаже)
- volatility (oscillations)

## 5. ГИПОТЕЗЫ ПО ЛОГИКЕ БОТА

### Entry logic (предполагаемая):
```python
if (bc_real_sol < 5.0 AND            # очень ранний вход
    dev_buy_sol > 0.5 AND            # дев вложил деньги
    n_buyers_in_first_3s >= 3 AND   # быстрый старт
    has_metadata == True):           # есть соцсети
    → ENTER
```

### Sizing logic (предполагаемая):
```python
score = (dev_buy_sol * w1 +
         n_early_buyers * w2 +
         buy_pressure * w3 +
         known_wallet_signal * w4)

if score > T6: size = 3.0 SOL
elif score > T5: size = 0.76 SOL
elif score > T4: size = 0.50 SOL
elif score > T3: size = 0.30 SOL
elif score > T2: size = 0.20 SOL
else: size = 0.09 SOL
```

### Re-buy logic (из данных):
- Если BC остается в диапазоне 10-22 SOL → продолжаем цикл buy/sell
- Если BC падает ниже точки входа → averaging down (до лимита)
- Если BC растет выше 25 SOL → take profit полная позиция

### Exit logic (из данных):
- Каждая покупка имеет соответствующую продажу (1:1 ratio)
- Продает по частям, не всю позицию сразу
- Останавливает торговлю на токене через 4-8 часов
